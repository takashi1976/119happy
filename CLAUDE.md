# 株式会社トミザワ LP プロジェクト方針

## プロジェクト概要
- 株式会社トミザワ様向けランディングページ制作プロジェクト。
- 技術スタックは `Astro` / `Tailwind CSS` / `Alpine.js` / `pnpm`。
- 公開先は `https://tomizawa2026.netlify.app` を想定。

## 現在の作業範囲
- 本番実装の前段として、開発環境、GitHub、Netlify の初期セットアップを優先する。
- 直近の制作着手は、LP全体のベースとなるワイヤーフレーム設計を最優先とする。
- Canva 由来のレイアウト、画像素材、テキストが揃い次第、本実装へ着手する。
- 現時点ではランディングページ本体のデザイン実装は未着手とする。

## 実装ルール
- ページは各セクションごとに必ずコンポーネント分割する。
- セクション右上にはデバッグモード用のコンポーネントファイルリンクを表示できる構成にする。
- Canva の書き出し素材とレイアウトを基準にしつつ、Astro コンポーネントへ適切に再構成する。
- セクション間の余白は全体で共通利用できるラッパーコンポーネントとして設計し、各セクションで再利用できる構成にする。
- 各セクションの見出しデザインは可能な限り共通コンポーネント化し、UI全体の統一感を担保する。
- 見出しは日本語のメインタイトルを主軸とし、英語のサブタイトルをアクセントとして併記する構成を基本とする。
- スタイルは `Tailwind CSS` を基本とし、状態制御や軽微なUI挙動は `Alpine.js` を利用する。
- 汎用レイアウト、セクション、デバッグUIは責務を分離して管理する。

## ワイヤーフレーム設計方針
- 初期段階では、実装より先にベースとなるワイヤーフレームを作成し、全体構成と各セクションの役割を整理する。
- `materials/screen-shot.png` は、顧客が Canva 上で各セクションを切り貼りして配置した参考イメージとして扱う。
- `materials/screen-shot.png` はあくまでラフ資料のため、そのまま再現するのではなく、構成や情報配置の参考として利用する。
- ワイヤーフレーム設計では、共通スペーシング、共通見出し、セクション分割ルールを先に固め、その後の本実装へスムーズに移行できる状態を目指す。
- 見出し構成の例として、「当社を選ぶ理由」のような日本語タイトルに対して `Reason` のような英語サブタイトルを添える設計を標準とする。

## ベンチマーク・参考デザイン
- ベンチマークサイトは `https://tokiwa-factory.com/` を参照する。
- ベンチマークサイトの参考ポイントは、やわらかく親しみのある印象、余白を活かしたレイアウト、イラストや写真を織り交ぜた構成、太めのゴシック体による見出し表現とする。
- 参考LPのトンマナや情報配置は踏襲しつつ、株式会社トミザワ向けにグリーン中心の配色へ再構成する。
- Canva デザインの参照元は `https://www.canva.com/design/DAHBpRqFfdI/OEQh1LDUCZFOp4G8XTfmVQ/edit` とする。

## 顧客要望メモ
- メインカラーは濃いグリーンを強調色、薄いグリーンをアクセントカラーとして使用する。
- 背景色はグレージュを中心に、オフホワイトや薄いベージュも許容する。
- 全体の印象は、親しみがあり柔らかめの雰囲気を重視する。
- フォントはゴシック系でやや太めのウェイトを基本とする。
- 装飾はイラストやアイコンを多めに用いつつ、参考LPのように適度な余白を確保する。
- 暖色系ベースではなく、グリーンを中心とした色展開でデザインを組み立てる。

## 開発フロー
1. GitHub でリポジトリを作成する。
2. 実装内容をリポジトリへプッシュする。
3. Netlify へ接続し、デプロイする。

## 開発コマンド
- `pnpm dev`: ローカル開発サーバー起動
- `pnpm build`: 本番ビルド
- `pnpm preview`: ビルド結果のローカル確認
- `pnpm check`: Astro の型・整合性チェック
- `pnpm netlify:dev`: Netlify 環境でのローカル確認

## ディレクトリ運用方針
- `src/layouts`: 共通レイアウト
- `src/components`: セクション単位のUIコンポーネント
- `src/pages`: ルーティング対象のページ
- `src/styles`: グローバルスタイル
- `public`: 静的アセット
- `docs`: 代理店支給のCanva書き出し素材や補足資料の保管先

## 素材・テキスト資料について
- 添付ファイル内のテキスト情報を `CLAUDE.md` に反映する運用とする。
- 現時点では `docs` 配下に実ファイルは未配置のため、追加受領後に内容を追記する。
- Canva 書き出し画像、コピー、レイアウト指示、注記が届いたら最優先で反映する。
- `materials` 配下の参考素材についても、ワイヤーフレームおよび本実装の判断材料として適宜 `CLAUDE.md` に整理して反映する。
